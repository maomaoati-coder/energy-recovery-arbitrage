## The Problem Nobody Is Asking

Every performance benchmark in the semiconductor industry measures the same thing:

```
FLOPS / Watt
```

How much you compute per watt consumed.

Nobody measures what happens to those watts when computation ends. The answer is universal and unexamined: **they become heat, and they are discarded.**

This is not a minor inefficiency. This is the defining waste problem of the compute era.

## Where Does 100W Actually Go?

Trace a single watt from the grid to useful computation:

```
100W  ──── Grid Input
 │
 ├── −30W  Cooling overhead (PDU, CRAC units, chillers)
 │
 └── 70W   Reaches the server
      │
      ├── −25W  VRM conversion loss, board distribution
      │
      └── 45W   Reaches the chip
           │
           ├── −43W  Heat dissipation during computation
           │
           └── ~2W   Useful computational work


  2% end-to-end efficiency.
  0% of waste recovered.
  This is the industry standard.
```

## A New Metric

The current benchmark is structurally blind to recovery. Here is the metric that reflects true cost of ownership:

$$\text{FLOPS}_{net} = \frac{\text{TFLOPS}}{W_{consumed} - W_{recovered}}$$

This is **FLOPS / Watt-recovered** — net energy efficiency. It is the only single number that captures what a chip actually costs to run at scale.

## The Arbitrage

Energy recovery is not speculative. The thermal output of data centers is a known, stable, high-volume heat source. At industrial power rates ($0.07/kWh), partial recovery yields:

| Recovery Rate | Annual Value Unlocked |
|--------------|----------------------|
| 10% | **$1.12 billion / year** |
| 20% | **$2.24 billion / year** |
| 30% | **$3.36 billion / year** |
| 50% | **$5.60 billion / year** |
| 70% | **$7.84 billion / year** |

Applications for recovered thermal energy include district heating networks, industrial process heat, thermoelectric reconversion, and agricultural greenhouse climate systems. None of these require new physics. They require chip architectures that treat waste heat as an output product rather than an unavoidable byproduct.

## Why Nobody Does It

This is not a physics problem. It is an **incentives problem.**

```
┌─────────────────────────────┐        ┌──────────────────────────────┐
│      CHIP DESIGNER          │        │      DC OPERATOR             │
│                             │   ≠    │                              │
│  KPI: TFLOPS, bandwidth     │        │  KPI: PUE, OPEX, uptime      │
│  Paid to win benchmarks     │        │  Pays every watt consumed    │
│  Does not pay the bill      │        │  Cannot redesign the chip    │
│                             │        │                              │
│  → Optimizes peak perf.     │        │  → Absorbs all waste cost    │
└─────────────────────────────┘        └──────────────────────────────┘
                          ↑
               The gap between these two actors
               is where $11.2B in waste lives.
               That gap is the arbitrage.
```

The company that aligns chip architecture with total cost of ownership — not just benchmark scores — captures both markets simultaneously. **That company does not exist yet.**

## The Thermal Wall Is Coming

The AI compute race has a physical endpoint nobody on the roadmap is addressing.

- NVIDIA H100: 700W TDP per card
- NVL72 rack: ~120kW per rack
- A 10,000-GPU cluster: ~1.2 MW of heat generation
- Phoenix, AZ ambient summer temp: 45°C

You cannot cool this with air. You cannot build cooling infrastructure fast enough. You cannot keep doubling TDP every two years and call it a roadmap.

**The constraint is not algorithmic. It is thermodynamic.**

The winner of the next compute decade will not be the company that pushes more FLOPS through a thermal wall. It will be the company that **eliminates the wall.**
